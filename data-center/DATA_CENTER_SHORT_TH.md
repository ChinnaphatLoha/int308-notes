## Data Center คืออะไร
โกดังขนาดใหญ่ที่สร้างมาเพื่อเก็บ Server จำนวนมหาศาลให้ทำงานได้ตลอด 24 ชั่วโมง

จุดสำคัญ:
- ออฟฟิศธรรมดาเก็บ Server ไม่ได้ เพราะไฟตกนิดเดียวระบบก็พัง
- ต้องมีแอร์ควบคุมอุณหภูมิพิเศษ
- ข้อมูลข้างในสำคัญมาก จึงต้องมีระบบรักษาความปลอดภัยขั้นสูง

สิ่งที่ควรจำ:
- Data Center เหมือนห้อง ICU ของคอมพิวเตอร์ ขาดไฟหรือแอร์ไม่ได้เด็ดขาด

สรุป:
Data Center คือสถานที่ที่สร้างขึ้นเพื่อรักษาชีวิตของ Server ให้ทำงานได้ตลอดเวลา

## มาตรฐานการสร้าง (Standards)
การสร้าง Data Center ต้องมีมาตรฐานที่ชัดเจน ไม่สามารถสร้างเองตามใจชอบได้

จุดสำคัญ:
- TIA-942 คือพิมพ์เขียวหลัก บอกโครงสร้างและจุดวางสายไฟ
- ASHRAE TC 9.9 คือมาตรฐานเรื่องความเย็น อุณหภูมิ และความชื้น
- NFPA คือมาตรฐานเรื่องการป้องกันไฟไหม้

สิ่งที่ควรจำ:
- ห้ามใช้น้ำดับไฟใน Data Center เด็ดขาด ต้องใช้ระบบดับเพลิงตามมาตรฐาน NFPA

สรุป:
การสร้างและการดูแลต้องอิง 3 มาตรฐานหลักคือ TIA (โครงสร้าง), ASHRAE (ความเย็น), และ NFPA (ดับเพลิง)

## ระดับความน่าเชื่อถือ (Tier System)
ระบบแบ่งเกรด Data Center ว่าทนต่อความเสียหายได้แค่ไหน

จุดสำคัญ:
- Tier I (Basic) มีสายไฟและแอร์ชุดเดียว พังคือล่มเลย
- Tier II (Redundant) มีอะไหล่สำรอง (เช่น UPS สำรอง) แต่ทางเดินไฟยังเส้นเดียว
- Tier III (Concurrently Maintainable) มีหลายเส้นทาง สามารถซ่อมบำรุงได้โดยที่ระบบไม่ล่ม
- Tier IV (Fault Tolerant) สลับเส้นทางได้อัตโนมัติในเสี้ยววินาที ระบบแทบไม่มีทางล่ม

สิ่งที่ควรจำ:
- สาเหตุที่ระบบล่มบ่อยที่สุด 60% มาจากการปิดซ่อมบำรุงตามแผนและปัญหา Software ไม่ใช่ Hardware พัง

สรุป:
ยิ่ง Tier สูงยิ่งแพงและเสถียร Tier III และ IV เหมาะกับระบบที่ห้ามล่มเด็ดขาด

## โครงสร้างห้อง (Physical Building)
การออกแบบภายในห้อง Server

จุดสำคัญ:
- ใช้พื้นยก (Raised Floor) เพื่อดันลมเย็นขึ้นมาจากใต้พื้น
- ซ่อนสายไฟและสายแลนไว้ใต้พื้นเพื่อความเป็นระเบียบ
- โครงสร้างเหล็กใต้พื้นช่วยป้องกันไฟฟ้าสถิต

สรุป:
พื้นยกคือหัวใจสำคัญในการจัดการลมเย็นและสายไฟในห้อง Server

## ระบบไฟฟ้า (Power)
ไฟคือสิ่งสำคัญที่สุด ไฟดับเท่ากับทุกอย่างจบ

จุดสำคัญ:
- Utility Power คือไฟหลวง ถือเป็นแหล่งพลังงานหลัก
- ATS คือสวิตช์ที่จะสลับไฟอัตโนมัติทันทีที่รู้ว่าไฟหลวงดับ
- UPS คือแบตเตอรี่ที่จ่ายไฟทันทีเพื่อยื้อเวลาไว้ 5-30 นาที
- Generator คือเครื่องปั่นไฟที่จะทำงานต่อจาก UPS จนกว่าน้ำมันจะหมด

สิ่งที่ควรจำ:
- ลำดับการทำงานเมื่อไฟดับคือ ไฟหลวงดับ > UPS จ่ายไฟแทน > เครื่องปั่นไฟติด

สรุป:
ระบบไฟต้องทำสำรองเป็นทอด ๆ เพื่อรับประกันว่า Server จะไม่ดับแม้แต่วินาทีเดียว

## ระบบทำความเย็น (Cooling)
Server ร้อนมาก จึงต้องมีระบบแอร์เฉพาะทาง

จุดสำคัญ:
- ใช้แอร์สำหรับ Data Center (PAC) เน้นดึงความร้อนออกและคุมความชื้น (แอร์ทั่วไปเน้นดึงความชื้น)
- CRAC ใช้น้ำยาแอร์ ส่วน CRAH ใช้น้ำเย็นจากภายนอก
- จัดตู้ Server หันหน้าเข้าหากัน (Cold Aisle) เพื่อดูดลมเย็น และหันหลังเข้าหากัน (Hot Aisle) เพื่อปล่อยลมร้อน

สิ่งที่ควรจำ:
- PUE คือค่าชี้วัดความประหยัดพลังงาน ยิ่งเข้าใกล้ 1.0 ยิ่งดี (แสดงว่าไม่เปลืองไฟไปกับระบบแอร์)

สรุป:
ต้องใช้ PAC คุมทั้งความร้อนและความชื้น และจัดทางเดินลมเย็นลมร้อนไม่ให้ปนกัน

## ระบบดับเพลิง (Fire Protection)
ระบบตรวจจับและดับไฟที่ปลอดภัยต่ออุปกรณ์อิเล็กทรอนิกส์

จุดสำคัญ:
- VESDA เป็นระบบดูดอากาศไปวิเคราะห์ รู้ล่วงหน้าก่อนเกิดควันไฟ
- Clean Agent เป็นแก๊สดับไฟ (เช่น FM-200, Novec 1230) ที่ดึงความร้อนออกและระเหยไปเอง

สิ่งที่ควรจำ:
- แก๊สพวกนี้ดับไฟได้โดยไม่ทิ้งคราบและไม่ทำให้อุปกรณ์พัง

สรุป:
ตรวจจับด้วย VESDA และดับด้วยแก๊ส Clean Agent เพื่อรักษา Server ให้ปลอดภัย

## ความปลอดภัย (Security)
การป้องกันการบุกรุกจากบุคคลภายนอก

จุดสำคัญ:
- มีการป้องกันหลายชั้นตั้งแต่ รปภ. รั้ว และกล้องวงจรปิด
- ใช้คีย์การ์ดและการสแกนลายนิ้วมือหรือม่านตา
- มี Mantrap (ประตู 2 ชั้น) ต้องปิดประตูนอกก่อน ประตูในถึงจะเปิด เพื่อกันคนเดินตาม

สรุป:
ความปลอดภัยทางกายภาพต้องทำเป็นชั้น ๆ เพื่อให้เจาะเข้ามาถึงตัว Server ได้ยากที่สุด

## การมอนิเตอร์ระบบ (Monitoring)
การเฝ้าระวังระบบทั้งหมด

จุดสำคัญ:
- NOC คือห้องควบคุมที่มีเจ้าหน้าที่เฝ้าดูหน้าจอ 24 ชั่วโมง
- DCIM คือซอฟต์แวร์ที่ใช้ดูภาพรวมของทั้ง Data Center เช่น การใช้ไฟและอุณหภูมิ

สรุป:
ต้องมีทั้งคน (NOC) และซอฟต์แวร์ (DCIM) คอยดูระบบไม่ให้คลาดสายตา

## แผนกู้ภัยเมื่อเกิดภัยพิบัติ (Disaster Recovery)
การเตรียมศูนย์สำรองเมื่อศูนย์หลักพัง

จุดสำคัญ:
- ระยะห่างของศูนย์สำรองควรอยู่ประมาณ 120-160 กม. เพื่อให้พ้นจากภัยพิบัติเดียวกัน แต่เน็ตยังส่งข้อมูลได้ไว
- Cold Site เป็นห้องเปล่า (กู้ระบบเป็นสัปดาห์)
- Warm Site มีเซิร์ฟเวอร์รอไว้แล้ว (กู้ระบบเป็นวัน)
- Hot Site เหมือนศูนย์หลักทุกอย่าง (กู้ระบบในหลักนาที)
- การ Sync ข้อมูลมีแบบ Synchronous (พร้อมกัน ไม่เสียข้อมูล) และ Asynchronous (ดีเลย์นิดหน่อย ส่งได้ไกลกว่า)

สิ่งที่ควรจำ:
- RTO คือคำถามว่าระบบล่มได้นานแค่ไหน (เน้นเวลา)
- RPO คือคำถามว่ายอมข้อมูลหายได้มากแค่ไหน (เน้นปริมาณข้อมูล)

สรุป:
DR คือแผนสำรองเมื่อเกิดเหตุไม่คาดฝัน โดยมี RTO และ RPO เป็นตัวกำหนดว่าต้องใช้ศูนย์สำรองแบบไหน
