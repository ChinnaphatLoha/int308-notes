## การเก็บและจัดการข้อมูล (Database)
ระบบที่ใช้จัดเก็บ ค้นหา และดูแลข้อมูลทั้งหมดของแอปพลิเคชัน

จุดสำคัญ:
- RAID คือการนำฮาร์ดดิสก์หลายลูกมามัดรวมกัน ฮาร์ดดิสก์พัง 1 ลูก ข้อมูลก็ไม่หาย
- DBMS คือโปรแกรมจัดการฐานข้อมูล ทำหน้าที่เหมือนบรรณารักษ์หาหนังสือให้เรา
- Concurrent Access คือการที่ DBMS ช่วยล็อกข้อมูลเมื่อมีคนแก้ไขพร้อมกัน เพื่อไม่ให้ข้อมูลเพี้ยน
- Relational Database เก็บข้อมูลเป็นตาราง มี Primary Key ระบุตัวตนห้ามซ้ำ และ Foreign Key ใช้เชื่อมกับตารางอื่น

สิ่งที่ควรจำ:
- View คือผลลัพธ์จำลองที่ดึงมาเฉพาะบางคอลัมน์ ใช้บังข้อมูลสำคัญเพื่อความปลอดภัยได้

สรุป:
ข้อมูลที่สำคัญต้องเก็บผ่านระบบ DBMS บนฮาร์ดดิสก์ที่ทำ RAID ไว้เพื่อป้องกันความเสียหาย

## การควบคุมสิทธิ์การเข้าถึง (Access Control)
การกำหนดว่าใครสามารถทำอะไรกับข้อมูลได้บ้าง

จุดสำคัญ:
- Authorization Table ทำหน้าที่เหมือนรายชื่อแขก VIP เอาไว้เช็กสิทธิ์
- การตั้งกฎมีหลายแบบ เช่น แอดมินกลางคุม (Centralized) หรือเจ้าของข้อมูลคุมเอง (Ownership-based)
- คำสั่งจัดการสิทธิ์ใน SQL คือ GRANT (ให้สิทธิ์) และ REVOKE (ดึงสิทธิ์คืน)

สิ่งที่ควรจำ:
- RBAC (Role-Based Access Control) คือการให้สิทธิ์ผ่านตำแหน่ง สำคัญตรงที่จัดการง่ายเวลาย้ายแผนกหรือพนักงานลาออก

สรุป:
การคุมสิทธิ์ที่ดีควรใช้ RBAC เพื่อลดความผิดพลาดและจัดการได้รวดเร็ว

## การโจมตีด้วย SQL Injection (SQLi)
การที่แฮกเกอร์พิมพ์คำสั่ง SQL แทรกเข้ามาในช่องกรอกข้อมูล เพื่อหลอกให้ระบบทำงานตามคำสั่งนั้น

จุดสำคัญ:
- Inband คือการหลอกแล้วเห็นผลลัพธ์ตรง ๆ บนหน้าเว็บ
- Inferential (Blind) คือการเดาจากการเปลี่ยนแปลงเล็ก ๆ เช่น เวลาโหลดหน้าเว็บ
- Out-of-band คือการบังคับให้ Database แอบส่งข้อมูลออกไปที่อื่น

สิ่งที่ควรจำ:
- วิธีป้องกันที่สำคัญที่สุดคือ Parameterized Queries ซึ่งเป็นการแยกคำสั่งกับข้อมูลออกจากกันอย่างเด็ดขาด

สรุป:
SQLi เป็นการโจมตีที่อันตรายมาก แต่ป้องกันได้ง่าย ๆ ด้วยการเขียนโค้ดแบบ Parameterized Queries เสมอ

## การเข้ารหัสฐานข้อมูล (Database Encryption)
การทำข้อมูลให้อ่านไม่ออกหากไม่มีกุญแจถอดรหัส

จุดสำคัญ:
- ข้อดีคือปลอดภัยสูงสุด แม้แฮกเกอร์ขโมยไฟล์ไปได้ก็อ่านไม่ออก
- ข้อเสียคือระบบจะทำงานช้าลงมากในการค้นหาข้อมูล และถ้ากุญแจหาย ข้อมูลจะหายถาวร

สิ่งที่ควรจำ:
- โดยทั่วไปจะไม่เข้ารหัสทั้งก้อน แต่จะเลือกเข้ารหัสเฉพาะข้อมูลที่อ่อนไหว เช่น รหัสผ่าน หรือเลขบัตรเครดิต

สรุป:
การเข้ารหัสมีต้นทุนเรื่องความเร็ว จึงต้องเลือกใช้เฉพาะกับข้อมูลที่สำคัญจริง ๆ เท่านั้น

## ความปลอดภัยบนคลาวด์ (Cloud Security)
การนำข้อมูลและระบบไปฝากไว้บนเซิร์ฟเวอร์ของผู้ให้บริการอินเทอร์เน็ต

จุดสำคัญ:
- Cloud มีลักษณะเด่น 5 อย่าง คือ ใช้ทรัพยากรร่วมกัน กดสร้างเองได้ ยืดหยุ่นขยายอัตโนมัติ จ่ายเท่าที่ใช้จริง และเข้าถึงได้ทุกที่
- IaaS คือการเช่าเครื่องเปล่า PaaS คือเช่าที่รันโค้ด SaaS คือเช่าแอปสำเร็จรูป (เช่น Gmail)
- Public Cloud ใช้ร่วมกับคนอื่น (ราคาถูก) Private Cloud ใช้คนเดียว (ปลอดภัย) Hybrid คือผสมกัน

ความต่างเรื่องรูปแบบฐานข้อมูลบนคลาวด์:
- แบบ Multi-instance (เหมือนบ้านเดี่ยว) ปลอดภัยกว่าแต่ราคาแพง
- แบบ Multi-tenant (เหมือนคอนโด) แชร์ระบบกัน ประหยัดกว่า แต่เสี่ยงข้อมูลหลุดไปหาคนอื่นถ้าซอฟต์แวร์มีช่องโหว่

สิ่งที่ควรจำ:
- SECaaS (Security as a Service) คือการเช่าระบบรักษาความปลอดภัยผ่านคลาวด์ เช่น Cloud Antivirus โดยไม่ต้องซื้อเครื่องมาตั้งเอง

สรุป:
คลาวด์ช่วยให้ทำงานง่ายและประหยัด แต่ต้องเข้าใจความเสี่ยงเรื่องการแชร์พื้นที่และระบบป้องกันที่ผู้ให้บริการเตรียมไว้ให้
